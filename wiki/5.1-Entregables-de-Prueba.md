# 5.1 Entregables de Prueba

Para el proceso de pruebas unitarias de `alf.io`, se generarán los siguientes artefactos como evidencia del cumplimiento de los objetivos de calidad, integrando tanto el backend como el frontend. Los entregables se ubicarán en la sección "Informe de Pruebas Unitarias" de esta Wiki.

- **Reporte de Ejecución de Pruebas Unitarias (Backend):** Resultados detallados de éxito/falla generados por JUnit para la lógica en Java.
- **Reporte de Ejecución de Pruebas Unitarias (Frontend):** Resultados de las pruebas de componentes y servicios generados por Karma y Jasmine.
- **Reporte de Cobertura de Código Agregado:** Evidencia técnica consolidada que demuestre que se ha alcanzado el 85% de cobertura total (agregando backend y frontend) utilizando herramientas como JaCoCo (para Java) y el reporter de cobertura integrado en Karma (para Angular).
- **Matriz de Trazabilidad:** Documento que vincula los casos de prueba unitarios con los requisitos funcionales del sistema. 
- **Lista de Defectos Encontrados:** Registro de todos los bugs identificados durante la ejecución de las pruebas unitarias, detallando su severidad y estado.
